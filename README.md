# 好厝邊 Haocuobian 應用程式

在這個數位時代，一句溫暖的問候，更能拉近彼此的距離。「好厝邊」是一款專為您設計的暖心問候圖製作工具，讓您能輕鬆地將關懷與祝福，透過精美的圖片傳遞給生命中重要的親朋好友。

無論是每日的早安、晚安，還是特殊節日的祝福，這裡都有豐富的素材和強大的工具，讓您的心意完美呈現。

## ✨ 核心功能 (Core Features)

*   **豐富圖庫**：內建「福氣花開」、「寶島風光」等多樣化圖庫，並依節日提供專屬祝福圖，靈感源源不絕。
*   **個人化創作**：支援上傳您自己的照片，讓問候圖更具個人風格與紀念意義。
*   **強大編輯器**：
    *   **祝福語庫**：內建超過10種分類的祝福語，一鍵即可取用。
    *   **專業字體**：提供多款精選中文字體，包含典雅的宋體與手寫感的書法字體。
    *   **完整客製化**：自由調整文字顏色、大小、角度、粗體、陰影及外框。
    *   **直式書寫**：支援文字直式/橫式切換，滿足傳統書寫美感。
*   **作品管理**：自動保存您的創作於「我的作品」，並可將喜愛的背景圖加入「我的最愛」，方便隨時再次編輯或分享。
*   **遊戲化系統**：透過創作、分享、完成任務來累積「暖心點數」、提升等級、解鎖各式各樣的成就徽章。
*   **一鍵分享**：完成後可直接儲存圖片至裝置，或透過系統原生介面輕鬆分享給好友。

## 🏗️ 系統架構與技術棧 (System Architecture & Tech Stack)

本專案採用現代化的前端技術棧，以元件化與關注點分離的原則進行開發，確保程式碼的可讀性與可維護性。

#### **技術棧 (Tech Stack)**

-   **核心框架**: React 18 & TypeScript
-   **圖片編輯**: Fabric.js
-   **建置工具**: Vite
-   **原生打包**: Capacitor
-   **樣式管理**: CSS Variables & Global Stylesheet

#### **檔案結構 (File Structure)**

-   **`src/`**: 專案原始碼主目錄。
    -   **`components/*.tsx`**: 可重用的 UI 元件 (如導覽列 `BottomNavBar`、彈出視窗 `SuccessModal`)。
    -   **`screens/*.tsx`**: 各個獨立的畫面 (如 `HomeScreen`, `EditorScreen`, `ProfileScreen`)。
    -   **`contexts/AppContext.tsx`**: React Context，集中管理並提供全域狀態與函式。
    -   **`hooks/*.ts`**: 封裝複雜 logique 的自訂鉤子。
        -   `useNavigation.ts`: 管理頁面切換、彈窗等導覽狀態。
        -   `useUserProfile.ts`: 管理使用者資料、成就和任務系統。
        -   `useStorage.ts`: 簡化與瀏覽器 LocalStorage 的互動。
    -   **`data.ts`**: 應用程式的「大腦」，定義所有靜態資料，包括圖庫、祝福語、遊戲化系統規則等。
    -   **`types.ts`**: 全域 TypeScript 型別定義。

## 🖼️ 自訂您的圖庫 (Customizing Your Image Library)

您可以輕鬆地新增或替換 App 內的圖庫內容。

#### **第一步：準備您的圖片與資料夾**

1.  在專案的根目錄下，找到 `public/` 資料夾。
2.  在 `public/` 內，建立一個名為 `images/` 的資料夾。
3.  根據 `data.ts` 中的設定，在 `images/` 資料夾中建立對應的子資料夾，並放入您的圖片。

一個與 `data.ts` 檔案完全對應的完整資料夾結構範例如下：

```
public/
└── images/
    ├── flowering_bliss/
    │   ├── f1.webp
    │   └── ... (依 data.ts 設定)
    │
    └── ... (其他分類資料夾)
```

#### **第二步：更新圖片清單 (`data.ts`)**

圖片的分類與路徑統一由 `data.ts` 檔案管理。該檔案使用輔助函式來自動生成路徑，讓管理大量圖片變得非常簡單。

**範例：若要將「福氣花開」分類的圖片數量從 47 張擴充到 50 張**

您需要：
1.  將 `f48.webp`, `f49.webp`, `f50.webp` 三個圖片檔放到 `public/images/flowering_bliss/` 資料夾中。
2.  修改 `data.ts` 中對應的 `generatePaths` 函式：

```typescript
// data.ts

export const CATEGORIES = {
  // 將 generatePaths 的第三個參數從 47 改為 50
  '福氣花開': createImagePaths(
    '/images/flowering_bliss', 
    generatePaths('f', 1, 50) // <-- 修改這裡
  ),
  // ... 其他分類
};
```
修改完成並儲存後，重新啟動您的應用程式，新的圖片就會自動出現在圖庫中。

## 📱 行動裝置 App 建置指南 (Capacitor)

本專案已整合 Capacitor，可輕鬆將此網頁應用程式打包成 iOS 和 Android App。

### **前置準備 (Prerequisites)**

在開始之前，請確保您的開發環境已安裝以下工具：
1.  **Node.js**: [下載連結](https://nodejs.org/)
2.  **iOS 開發**: macOS 電腦, [Xcode](https://developer.apple.com/xcode/), [CocoaPods](https://cocoapods.org/)
3.  **Android 開發**: [Android Studio](https://developer.android.com/studio)

### **開發流程**

**第一步：安裝專案依賴**
```bash
npm install
```

**第二步：新增原生平台 (僅需執行一次)**
```bash
npx cap add ios
npx cap add android
```

### **App 圖示與啟動畫面 (Icon and Splash Screen)**

本專案使用 Capacitor 官方工具 `@capacitor/assets` 來自動產生所有圖示與啟動畫面，這能確保最佳的相容性。

**1. 安裝工具 (僅需執行一次)**
```bash
npm install -D @capacitor/assets
```

**2. 準備素材圖片**
在專案的**根目錄**下，建立一個名為 `resources` 的資料夾，並放入以下素材：
-   `icon.png`: 您的 App 圖示，**必須是方形**，建議尺寸 `1024x1024` 像素。
-   `splash.png`: 您的 App 啟動畫面，建議尺寸 `2732x2732` 像素。
-   **(可選) `icon-foreground.png`**: 專為 Android 自適應圖示設計的前景層。
-   **(可選) `icon-background.png`**: 專為 Android 自適應圖示設計的背景層（例如色塊）。

**3. 自動產生與複製**
每當您更新了 `resources` 資料夾中的素材後，執行以下單一指令：
```bash
npx @capacitor/assets generate
```
此工具會自動將圖片裁切成所有需要的尺寸，並複製到 iOS 和 Android 的專案資料夾中。

### **建置與同步**

**1. 建置網頁應用程式**
每次更新網頁程式碼後，都需要重新建置。
```bash
npm run build
```

**2. 同步網頁資源到原生平台**
此指令會將最新的網頁內容和資源 (包含圖示) 複製到原生專案中。
```bash
npm run sync
```

**3. 在 IDE 中開啟並執行**
*   **iOS**:
    ```bash
    npm run open:ios
    ```
    此指令會自動在 Xcode 中開啟專案，接著您便可執行到模擬器或實體裝置上。

*   **Android**:
    ```bash
    npm run open:android
    ```
    此指令會自動在 Android Studio 中開啟專案，等待 Gradle 同步後即可執行。
